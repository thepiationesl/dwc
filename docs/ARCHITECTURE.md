## 核心模块设计

### 1. 容器入口模块 (entrypoint.sh)

所有容器共享的入口脚本，负责：

1. **用户初始化**
   - 创建用户和组
   - 设置用户密码
   - 配置 sudo 权限

2. **目录配置**
   - 创建配置目录
   - 设置目录权限
   - 复制配置模板

3. **服务启动**
   - 启动 VNC 服务器
   - 启动 noVNC 代理
   - 启动音频桥接服务

```bash
#!/bin/bash
# entrypoint.sh 主要流程

# 1. 用户初始化
init_user() {
    useradd -m -u 1000 -s /bin/bash qwe
    echo "qwe:toor" | chpasswd
    usermod -aG sudo qwe
}

# 2. 目录配置
setup_directories() {
    mkdir -p /config
    cp -r /etc/skel/. /config/
    chown -R qwe:qwe /config
}

# 3. 权限修复
fix_permissions() {
    /usr/local/bin/fix-permissions.sh
}

# 4. 服务启动
start_services() {
    # 启动 VNC
    su - qwe -c "vncserver :1 -geometry 1920x1080 -depth 24"
    
    # 启动 noVNC
    /usr/share/novnc/utils/novnc_proxy --vnc localhost:5901 --listen 6080
    
    # 启动音频桥接（如果支持）
    if [ -f /usr/local/bin/audio-bridge.sh ]; then
        /usr/local/bin/audio-bridge.sh &
    fi
}
```

### 3. noVNC 自定义模块

增强的 noVNC 实现，支持中文界面和音频控制：

```javascript
// audio-client.js 核心功能
class AudioClient {
    constructor() {
        this.audioContext = null;
        this.scriptNode = null;
        this.socket = null;
    }
    
    connect(port = 6081) {
        this.socket = new WebSocket(`ws://${window.location.hostname}:${port}`);
        this.socket.binaryType = 'arraybuffer';
        
        this.socket.onmessage = (event) => {
            this.playAudio(new Float32Array(event.data));
        };
    }
    
    playAudio(audioData) {
        if (!this.audioContext) {
            this.audioContext = new (window.AudioContext || window.webkitAudioContext)();
        }
        
        const buffer = this.audioContext.createBuffer(1, audioData.length, 44100);
        buffer.getChannelData(0).set(audioData);
        
        const source = this.audioContext.createBufferSource();
        source.buffer = buffer;
        source.connect(this.audioContext.destination);
        source.start();
    }
}
```

## 架构模式

### 1. 分层架构

```
┌─────────────────────────────────────────────────────────────┐
│                   应用层 (Application Layer)                 │
│         桌面环境、应用程序、用户界面                         │
├─────────────────────────────────────────────────────────────┤
│                   服务层 (Service Layer)                     │
│         VNC、音频、网络、文件共享                           │
├─────────────────────────────────────────────────────────────┤
│                   运行时层 (Runtime Layer)                   │
│         容器运行时、虚拟化、系统服务                         │
├─────────────────────────────────────────────────────────────┤
│                   基础设施层 (Infrastructure Layer)          │
│         Docker、Linux 内核、硬件资源                        │
└─────────────────────────────────────────────────────────────┘
```

### 2. 微服务模式

每个容器作为独立的微服务：

- **单一职责** - 每个容器专注于特定功能
- **独立部署** - 容器可以单独部署和更新
- **松耦合** - 通过接口和协议通信
- **可扩展** - 可以水平扩展特定服务

### 3. 配置即代码

所有配置通过代码管理：

```yaml
# docker-compose.yml 示例
version: "3.8"
services:
  debian-desktop:
    build:
      context: .
      dockerfile: desktop/debian/Dockerfile
    image: dwc/debian-desktop:latest
    volumes:
      - ./data/debian-desktop:/config
    ports:
      - "6080:6080"
    environment:
      - USERNAME=qwe
      - PASSWORD=toor
    restart: unless-stopped
```

                │

                │

```

          (音频输出)    (PCM编码)    (实时传输)   (Web Audio)
```

## 安全架构

### 1. 容器安全

```dockerfile
# 安全最佳实践
FROM alpine:edge

# 使用非 root 用户
RUN addgroup -g 1000 appgroup && \
    adduser -u 1000 -G appgroup -s /bin/sh -D appuser

# 设置适当的权限
RUN chmod 755 /app && \
    chown -R appuser:appgroup /app

# 使用只读文件系统（部分目录除外）
VOLUME ["/config", "/tmp"]

USER appuser
```

### 2. 网络安全

- **端口最小化** - 只暴露必要端口
- **防火墙规则** - 使用 iptables 限制访问
- **SSL/TLS** - 生产环境使用加密传输

### 3. 数据安全
